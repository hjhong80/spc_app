# 커밋 접두사 설명 문서 설계

## 작업 목적
- GitHub 업로드와 CI/CD 준비 전에 자주 쓰는 커밋 메시지 접두사를 한눈에 볼 수 있게 정리한다.
- 팀/개인 작업에서 흔히 쓰는 Conventional Commits 계열 접두사의 의미를 한국어로 설명한다.

## 설계
- 설명 대상은 `feat`, `fix`, `docs`, `style`, `refactor`, `test`, `chore`, `build`, `ci`, `perf`, `revert`로 제한한다.
- 문서는 `.docs/2026-03-08` 아래에 저장해 작업 메모 성격을 유지한다.
- 각 항목마다 의미, 언제 쓰는지, 짧은 예시를 함께 적는다.

## 검증 기준
- 문서가 한국어로 작성되어 있어야 한다.
- 사용 빈도가 높은 접두사가 빠지지 않아야 한다.
- 예시가 실제 커밋 메시지 형태로 바로 참고 가능해야 한다.
