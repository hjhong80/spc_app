# 세부 차트 정규분포 레이아웃 설계

## 작업 목적
- 세부 월/일 차트에 정규분포 보기 모드를 추가한다.
- 기존 세부 월 캔들, 세부 일 꺾은선은 그대로 유지한다.
- 정규분포용 raw 측정값은 새 백엔드 엔드포인트로 기간 조건 조회해 성능 저하를 방지한다.

## 현재 구조

- 세부 차트 컨테이너는 [`DetailCharacteristicChart.jsx`](/c:/Users/USER/Documents/MyProjects/spc_project/spc_front/src/component/chart/DetailCharacteristicChart.jsx)에서 우측 상단 `overlayControls`를 이미 지원한다.

## UI 설계

### 1. 상단 토글
- 우측 상단에 1차 토글을 둔다.
- 세부 월 차트: `캔들(기본)` / `정규분포`
- 세부 일 차트: `꺾은선` / `정규분포(기본)`
- 1차 토글 두 버튼은 동일한 폭으로 맞춘다.
- 점 표시 모드(`러그 점/곡선 정렬점`)는 기능만 유지하고 차트 UI 버튼은 숨긴다.
- 일방공차 곡선 타입(`단측 곡선/비대칭 곡선`)도 기능만 유지하고 차트 UI 버튼은 숨긴다.
- 현재 차트 UI 기본값은 `단측 곡선`이다.

### 2. 정규분포 뷰
- x축 중심은 `nominal`이다.
- 중심을 기준으로 좌우 8눈금을 사용한다.
- 6눈금은 공차 한계 정수 스케일이다.
- 7, 8눈금은 공차 밖 값에 대해 로그 압축한다.
- 메인 곡선은 가상 정규곡선이 아니라 `실측값 기반 KDE(커널 밀도 추정) 곡선`으로 렌더링한다.
- 실측값은 새 raw API가 반환한 개별 측정값 목록으로 점을 렌더링한다.
- 얇은 세로 보조선으로 `nominal`과 `mean`을 함께 표시한다.
- 점 표현은 두 모드를 지원한다.
  - `곡선 정렬점`: 각 실측값 x 위치에서 이론 곡선 y 값에 붙여 표시
  - `러그 점`: 각 실측값 x 위치에 바닥 근처 지터 점으로 표시
- `nominal=0`, `lTol=0`, `uTol>0`이면 무조건 일방공차로 간주한다.
- 이 경우 정규분포 해석은 `0`에 가까울수록 양호한 값으로 본다.
- 현재 표시 중인 정규분포 데이터에 음수 실측값이 1개라도 있으면 노이즈로 판단한다.
- 노이즈가 감지되면 헤더 우측에 붉은색 경고 아이콘을 표시하고, 클릭 시 경고 메시지를 읽을 수 있어야 한다.
- 일방공차 해석 문구와 노이즈 경고는 유지하되, 실제 곡선 자체는 측정값 기반 KDE를 우선 사용한다.

## 백엔드 설계

### 1. 새 엔드포인트
- 새 엔드포인트를 추가한다.
- 예시 경로: `/spcdata/report/project/{projId}/characteristic/{charId}/distribution`
- 파라미터:
  - `scale=month|day`
  - `baseDate=YYYY-MM|YYYY-MM-DD`

### 2. 조회 방식
- 전체 이력을 조회한 뒤 자르지 않는다.
- DB mapper에서 `charId + 기간` 조건으로 직접 raw 측정값만 조회한다.
- 월 모드:
  - 해당 `YYYY-MM` 범위의 측정값 목록
- 일 모드:
  - 해당 `YYYY-MM-DD` 범위의 측정값 목록
- 함께 내려줄 메타:
  - `nominal`, `uTol`, `lTol`
  - `mean`, `sigma`, `min`, `max`
  - `sampleCount`
  - `measuredValues`

## 프론트 데이터 흐름
- 세부 기본 차트 데이터는 기존 detail API를 그대로 사용한다.
- 정규분포 토글 진입 시 distribution API를 호출한다.
- 같은 `charId + scale + baseDate` 조합은 상태에 캐시한다.
- 정규분포 모드에서만 distribution API 에러/로딩 메시지를 분리해 보여준다.

## 차트 좌표 설계
- x축:
  - raw 측정값을 nominal 중심 상대 좌표로 변환한다.
  - 공차 내부 0~6 구간은 선형
  - 공차 외부 6~8 구간은 로그 압축
- y축:
  - 정규분포 밀도값 기준 상대 높이
  - 러그 점 모드에서는 바닥 근처 작은 y 분산을 허용
- 툴팁:
  - `측정값`, `nominal 대비 편차`, `공차 대비 위치`, `표준편차`, `선택 기간 샘플 수`

## 세부 월/일 기본 차트 축 재설계

### 1. 변경 배경
- 기존 세부 월 캔들 / 세부 일 꺾은선은 `공차 대비 편차(%)` 축을 내부 좌표로 사용한다.
- Y축 라벨만 실측값으로 바꾸면, 화면에 보이는 값과 실제 좌표계가 달라져 툴팁, 기준선, 로그 압축 해석이 어긋날 수 있다.
- 따라서 세부 월/일 기본 차트는 라벨만 바꾸지 않고, 내부 좌표계 자체를 `실측값 기반 축`으로 재구성한다.

### 2. 새 축 기준
- 중심값 `center = (USL + LSL) / 2`
- `display = 0`은 중심값 `center`
- `display = -100`은 `LSL`
- `display = +100`은 `USL`
- `LSL ~ center ~ USL` 구간은 실측값 기준 선형 매핑
- 공차 바깥 구간은 기존과 같은 로그 압축을 유지한다.

### 3. 로그 압축 규칙
- 공차 내부:
  - `LSL <= raw <= USL` 구간은 선형
- 공차 외부:
  - `raw < LSL` 또는 `raw > USL` 구간은 기존 `100 초과 영역 압축` 개념을 유지한다.
  - 내부 display 개념은 유지하되, 사용자에게 보이는 라벨과 툴팁은 항상 실측값으로 제공한다.
- 즉 시각적 거리감은 기존과 유사하게 유지하고, 축 해석만 실측값 일관성으로 바꾼다.

### 4. 적용 범위
- 세부 월 차트 캔들 Y축
- 세부 일 차트 꺾은선 Y축
- 세부 기본 차트 툴팁의 값 표시
- 기준선(`LSL`, `USL`, 중심선) 위치 계산
- 확대/스크롤 이후에도 동일 좌표계를 유지한다.

### 5. 구현 원칙
- 기존 `toTolerancePercent(...)` 기반 좌표 계산은 세부 기본 차트 경로에서 제거한다.
- 대신 `toDetailDisplayValue(raw)` / `toDetailRawValue(display)` 쌍을 새로 둔다.
- 축 formatter는 내부 display 값이 아니라 역변환된 실측값을 표시한다.
- 캔들 body, wick, line point, hover tooltip 모두 같은 변환기를 사용한다.
- 정규분포 차트는 이미 실측값 기반 axisScale을 사용하므로, 이번 변경은 세부 기본 차트에만 적용한다.
