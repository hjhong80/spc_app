## 작업 목적
- 성적서 업로드/저장 흐름을 실제 API와 DB에 연결하고 고객 엑셀의 축 분리, 날짜/시간 분리, 좌표 표현 다양성을 흡수한다.
- 프로젝트 등록 UX와 업로드 성공/실패 피드백을 운영 가능한 수준으로 고도화한다.

## 주요 변경 사항
- 백엔드 업로드/저장 로직 구현
  - `/excel/upload`에서 파싱 후 `insp_report_tb`, `insp_data_tb` 저장까지 처리하도록 구현했다.
  - `measurement_time_source_json` 파싱을 다중 포맷과 AM/PM까지 처리하도록 강화했다.
  - `serial_no` 중복/필수값 검증과 저장 전 유효성 체크를 정리했다.
  - 업로드 중 `char_tb` 신규 생성 없이 기존 특성만 매칭하도록 고정했다.
- 특성 매칭 정책 개선
  - `char_no + axis + nominal` 우선 매칭을 적용했다.
  - axis가 없거나 매칭 실패 시 `char_no + nominal` fallback을 사용했다.
  - 중복 키는 스킵/차단되도록 보강했다.
- 스키마/모델 확장
  - `axis_col`(proj_tb), `axis`(char_tb)와 `measured_value_col` 연계를 전반에 반영했다.
  - axis nullable 정책을 적용해 필수 검증을 제거했다.
- SQL 및 운영 스크립트 정리
  - FK 제약 때문에 실패하던 TRUNCATE 문제를 해결할 스크립트를 정리했다.
  - axis 컬럼 추가 SQL을 MySQL 호환 방식으로 수정했다.
  - SQL 보관 경로 규칙을 날짜별 `sql` 디렉터리로 정리했다.
- 프론트 UX 개선
  - 프로젝트 매핑 필드에 `Axis (필요할 시)`를 추가했다.
  - 하이라이트 색상과 좌표 입력 규칙(`,`, `~`, `-`)을 프론트/백엔드에 동일하게 맞췄다.
  - 적용 성공 시 상단 알림, 버튼 비활성화, 2초 후 이전 페이지 이동 흐름을 추가했다.
