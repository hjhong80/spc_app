## 작업 목적
- 엑셀/프로젝트 데이터 처리 흐름을 백엔드 구조와 맞추고 프로젝트 등록/수정 UI를 실제 저장 가능한 상태로 정리한다.
- 프로젝트 파싱 설정(JSON)과 필드 매핑 결과를 프론트와 백엔드 DTO 계약으로 일치시킨다.
- 요청 바디 누락, DTO 타입 충돌, 실행 경로 불일치 등 연속 오류를 정리한다.

## 주요 변경 사항
- 프로젝트/엑셀 도메인 구조 정리
  - DTO 경로를 `dto/spcdata`로 정리했다.
  - `ProjectAddReqDto`, `ExcelInputReqDto`, `ProjectController`, `ExcelController`, `ProjectService`를 정리했다.
  - 프로젝트 CRUD와 권한 체크(`roleId >= 2`)를 서비스 계층에 반영했다.
- 프론트 프로젝트 등록/수정 UI 확장
  - 업로더/프로젝트 리스트 레이아웃을 넓히고 우측 패널 하단 정보 영역을 조건부 렌더링으로 변경했다.
  - `serialNumberSourceJson`, `measurementTimeSourceJson`을 폼 초기값과 프로젝트 선택 복원 흐름에 반영했다.
  - `DataSourceSelector` 상태를 JSON 문자열로 저장/복원하도록 만들었다.
- 프로젝트 저장 DTO/호출 정리
  - `buildProjectAddReqDto`로 백엔드 요청 DTO를 구성했다.
  - `postAddProject(dto)` 호출 시그니처에 맞춰 Orval 생성 API와 프런트 호출을 정리했다.
  - 디버그 로그와 DTO preview/response 로그를 유지했다.
- 백엔드 모델/매퍼 확장
  - `Project` 엔티티와 `ProjectAddReqDto`에 `serialNumberSourceJson`, `measurementTimeSourceJson`을 추가했다.
  - `project_mapper.xml` insert/select/update에 신규 컬럼을 반영했다.
  - `ProjectService`에 JSON 유효성 검증 로직과 한국어 메시지를 반영했다.
- 트러블슈팅 정리
  - VS Code launch `cwd`를 `spc_back`으로 맞췄다.
  - `Required request body is missing` 문제를 codegen 재실행과 API 호출 시그니처 정리로 해결했다.
  - `CharacteristicReqDto` 분리 후 구 inner class 타입 참조 충돌을 deprecated bridge로 완화했다.
