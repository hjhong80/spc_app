## 작업 목적
- SPC 프로젝트의 백엔드 업로드 기능과 기본 도메인 구조를 컴파일 가능한 상태로 정리한다.
- DB 스키마 기준 엔티티, MyBatis Mapper, Repository 계층을 맞추고 후속 서비스 구현 기반을 만든다.
- README와 개발 환경 설정을 정리해 테스트와 실행이 가능한 기본 골격을 갖춘다.

## 주요 변경 사항
- 백엔드 엑셀 업로드 기초 정리
  - 업로드 컨트롤러를 `multipart/form-data` 기준으로 정비했다.
  - `ExcelInputReqDto`와 `InspectionServiceImpl` 기초 구현을 맞추고 빈 주입 문제를 해결했다.
  - `/excel/**` 경로를 Swagger UI에서 테스트 가능하도록 permit 처리했다.
- 문서와 개발 환경 정리
  - 루트 `README.md`를 한글 기준으로 작성했다.
  - 루트 `.prettierrc`, `.vscode/settings.json`을 정리하고 저장 시 포맷 동작을 맞췄다.
  - 로컬 8080 포트 점유 프로세스를 확인하고 해제했다.
- DB/엔티티 정리
  - `proj_tb`, `char_tb`, `insp_report_tb`, `insp_data_tb` 기준 CREATE TABLE 초안을 정리했다.
  - 엔티티 `ProjTb`, `CharTb`, `InspReportTb`, `InspDataTb`를 추가했다.
- MyBatis/Repository 기초 구현
  - `Characteristic`, `Project`, `InspectionReport`, `InspectionData` Mapper와 XML을 생성했다.
  - `ProjectRepository`, `CharacteristicRepository`, `InspectionReportRepository`, `InspectionDataRepository`를 만들었다.
  - 표준편차 계산은 `STDDEV_POP` 기반 조건식으로 구현했다.
  - `char_tb`, `insp_report_tb`, `insp_data_tb` 실제 테이블명과 조회 성격에 맞게 SQL을 분리했다.
