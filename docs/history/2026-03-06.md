## 작업 목적
- 메인/세부 차트와 데이터 입력/조회 화면의 UX를 실사용 기준으로 정교화한다.
- 프로젝트 매퍼 수정 모드와 엑셀 미리보기 기반 입력 흐름을 안정화한다.
- 프론트 번들 구조와 백엔드 테스트 환경을 점검해 빌드/테스트 신뢰도를 높인다.

## 주요 변경 사항
- 차트 계산/표시 로직 고도화
  - nominal 기준 0%, 공차 대비 퍼센트 환산 규칙으로 차트를 정리했다.
  - 캔들 몸통은 `평균 ± 2σ`, 꼬리는 `min/max`를 사용하도록 반영했다.
  - 고정/동적 Y축 스케일, 35봉 슬롯 규칙, 가로 스크롤, 평균선/툴팁 표시를 반복 조정했다.
  - 메인 차트 캔들 -> 세부 월 -> 세부 일 전환 흐름을 단일 화면에서 재구성했다.
  - 세부 월/일 툴팁 정보, 색상 규칙, 헤더 칩과 등록 성적서 수 표시를 정리했다.
- 헤더/푸터/본문 규격 통일
  - `HeaderAuthStatus`, `UnifiedHeaderBar`, `UnifiedFooterBar`를 추가해 공통 레이아웃 구조를 맞췄다.
  - 홈/차트/데이터 입력/프로젝트 매퍼의 아이콘 간격, 패딩, 본문 `p:3`, `gap:2` 규격을 통일했다.
  - 차트의 정보 안내는 화면 고정형 배너에서 아이콘 앵커 팝오버로 변경했다.
- 데이터 입력/등록 UX 개선
  - 프로젝트 검색, 추천 프로젝트, 최근 사용 프로젝트, 핀 고정/해제를 구현했다.
  - 드래그앤드롭 존과 리스트 영역의 세로 균형을 조정했다.
  - 수정 모드에서는 기존 값 프리필과 변경 시에만 활성화되는 `수정` 버튼을 적용했다.
- 엑셀 업로드/파싱 보강
  - 분리된 `Date`/`Time` 셀과 시간 전용 숫자 셀 파싱 오류를 수정했다.
  - `Date + Time` 병합 문자열도 다시 추출하도록 보강했다.
  - `read-excel-file/browser` 전환 이후 샘플 파일 호환성을 확인했다.
- 번들/테스트 환경 정리
  - `react-query-devtools` dev 지연 로딩, 페이지 lazy 로딩, 청크 전략 단순화로 빌드 경고를 완화했다.
  - Maven Surefire에 Mockito `-javaagent`를 추가해 동적 에이전트 경고를 제거했다.
