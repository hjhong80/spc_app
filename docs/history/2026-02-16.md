## 작업 목적
- 데이터 입력/조회와 프로젝트 매퍼 흐름을 실사용 가능한 UX로 다듬고, 최근 사용 프로젝트와 추천 프로젝트 흐름을 강화한다.
- 차트/데이터 화면의 구조와 저장 API 흐름을 정비해 운영 가능한 상태로 만든다.
- 오탈자(`norminal`)와 BOM/인코딩 이슈를 제거해 유지보수 안정성을 높인다.

## 주요 변경 사항
- 데이터 입력/조회 화면 재구성
  - 업로드/프로젝트 목록/차트 중심 구조로 페이지를 정리했다.
  - 차트 관련 컴포넌트를 `component/spc-data` 계층 중심으로 리팩토링했다.
  - 공통 드롭존 `ExcelFileDropzone` 재사용 구조를 맞췄다.
- 프로젝트 추천/최근 프로젝트 UX 추가
  - 검색 인풋, 최근 사용 프로젝트 3개, 추천 프로젝트 3개 노출을 구현했다.
  - 최근 프로젝트는 로컬스토리지 기반으로 고정 표시하고 추천 프로젝트와 핀 상태를 함께 관리했다.
  - 파일 첨부 시 파일명 앞 5글자를 검색창에 자동 입력하도록 구성했다.
  - `useDebounce` 훅을 `src/utils/useDebounce.js`로 분리했다.
- 프로젝트 매퍼 UX 고도화
  - 필드 매핑 레이아웃을 메인/우측 패널 구조로 정리했다.
  - 엑셀 미리보기 클릭 기반 입력 흐름과 데이터 시작 행, 셀 좌표, 컬럼 문자 입력 규칙을 고정했다.
  - 프로젝트 선택 시 수정 모드로 전환하고 변경이 있을 때만 수정 버튼이 활성화되도록 했다.
  - 프로젝트 리스트 칩 표기 규칙을 `Start`, `Char`, `Axis`, `Nom`, `Act`, `UTol`, `LTol` 순으로 정리했다.
- 백엔드 조회/정비 작업
  - `norminal`을 `nominal`로 전역 정리했다.
  - BOM 파일을 전수 검사해 UTF-8(no BOM)으로 정리했다.
  - 최근 프로젝트 조회 API와 전용 DTO(`RecentlyProjectRespDto`)를 추가했다.
  - `selectRecentlyProjId` SQL과 `ReportService#getRecentlyProjectList()`를 구현했다.
- 업로드/차트 연동 정리
  - 세부 월/일 차트, 툴팁, 헤더 칩, 날짜 포맷 유틸을 정리했다.
  - 프로젝트 매퍼에서 `exceljs`를 `read-excel-file/browser`로 교체했다.
  - 미리보기 데이터의 `Date` 렌더 오류를 막기 위해 정규화 로직을 추가했다.
- 번들/테스트 환경 정리
  - `react-query-devtools`를 dev 전용 지연 로딩으로 전환했다.
  - 페이지 라우트를 lazy 로딩으로 바꾸고 Vite 청크 전략을 단순화했다.
  - Maven Surefire에 Mockito `-javaagent`를 추가해 동적 에이전트 경고를 정리했다.
